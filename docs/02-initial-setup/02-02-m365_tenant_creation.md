# 2.2 Microsoft 365 テナントの作成（教育機関向け）

## 📖 この節で学習すること

教育機関向けMicrosoft 365テナント（組織のクラウド専用スペース）の作成手順を、実際の画面操作と併せて詳しく解説します。テナント作成から初期管理者アカウントの設定、管理センターへのアクセスまでを網羅します。

> 💡 **重要**: テナントは教育機関のデジタル基盤となる重要な要素です。一度作成すると後から統合や移動が困難なため、慎重に設定を行ってください。

---

## 🎯 テナントとは何か

### **テナントの基本概念**

Microsoft 365の「テナント」とは、企業や大学などの組織がMicrosoftのクラウドサービスであるMicrosoft 365を利用するための「専用のスペース」に相当します。教育機関では、この専用スペース内で学校全体の情報システムを管理します。

```markdown
テナント = 教育機関がMicrosoftのクラウド上で借りている区画
```

### **教育機関におけるテナントの特徴**

#### **1. 独立した管理領域**
- **完全分離**: 組織ごとにファイルなどを保存する領域が割り当てられ、同じ組織のなかで情報を共有することができます
- **セキュリティ境界**: 他の教育機関とデータが混ざることは一切ありません
- **権限管理**: テナント内でのみ教職員・学生のアカウント管理が可能

#### **2. 教育機関特有の情報管理**
```markdown
教育機関テナントに含まれる主要情報:
├── 学校名・テナント名
├── ドメイン情報（onmicrosoft.com + 学校独自ドメイン）
├── 教育機関向けライセンス情報
├── 教職員・学生アカウント
├── クラス・グループ情報
└── 教育用セキュリティ設定
```

#### **3. 教育サービス連携の基盤**
- **統一認証**: 一度サインインで全Microsoft 365 Education サービス利用可能
- **教育データ共有**: クラス内でのシームレスな学習資料・課題共有
- **統合管理**: 一つの管理センターですべての教育サービスを管理

## 📋 テナント作成前の準備

### **3.1 教育機関として必要な情報・準備物**

#### **教育機関情報**
| 項目 | 詳細 | 例 |
|------|------|-----|
| **正式な教育機関名** | 文部科学省認可の正式名称 | ○○大学<br>○○市立△△小学校 |
| **教育機関種別** | 学校の種類 | 大学、高等学校、中学校、小学校 |
| **国・地域** | データセンターの場所に影響 | 日本 |
| **学校規模** | 教職員・学生数 | 教職員50名、学生500名 |
| **学校メールドメイン** | 既存のドメイン（ある場合） | school.edu.jp |

#### **管理者情報**
| 項目 | 詳細 | 注意点 |
|------|------|--------|
| **IT管理者氏名** | グローバル管理者の実名 | 正確な情報を入力 |
| **学校代表メール** | 認証用メールアドレス | 既存の学校メール |
| **電話番号** | 確認用電話番号 | SMS受信可能なもの |

#### **ライセンス調達準備**
- **CSP/LSPパートナー情報**: Microsoft 365 Education有償ライセンス購入先
- **予算承認書類**: A3/A5ライセンス購入時に必要
- **教育機関認証書類**: 学校法人証明書等

### **3.2 教育機関向けライセンス体系の理解**

#### **Microsoft 365 Education ライセンス概要**

| プラン | 取得方法 | 対象 | 主要機能 |
|--------|----------|------|----------|
| **Office 365 A1** | Microsoft直接申し込み（無料） | 教職員・学生 | Web版Office、Teams、OneDrive |
| **Microsoft 365 A1 for Device** | CSP/LSPパートナー | 共有デバイス | 基本的なデバイス管理機能とWindows Autopilotの一部機能を提供 |
| **Microsoft 365 A1 for Device (Legacy)** | CSP/LSPパートナー | 既存契約継続 | 限定的なデバイス管理機能とWindows Autopilotの基本的な利用のみ可能 |
| **Microsoft 365 A3** | CSP/LSPパートナー | 教職員・学生 | 完全なデバイス管理およびWindows Autopilotのフル機能やセキュリティ機能を提供 |
| **Microsoft 365 A5** | CSP/LSPパートナー | 教職員・学生 | 最上位プラン（高度分析・音声機能含む） |

#### **Student Use Benefit（学生使用特典）**
例えば、教職員1名と生徒50名の教育機関の場合、1名分のMicrosoft 365 A3 (Education Faculty Pricing) をご契約いただけば、生徒40名分Microsoft 365 A3 (Education Student Pricing) のライセンスが無料で利用できます

```markdown
Student Use Benefit の仕組み:
・教職員向けライセンス 1つにつき、学生向けライセンス 40個が無償提供
・全教職員がライセンス契約している場合のみ適用
・一部教職員のみの契約では適用外
```

### **3.3 テナント命名規則の検討**

#### **教育機関向け初期ドメイン（onmicrosoft.com）の決定**
```markdown
形式: [学校名].onmicrosoft.com

良い例:
- kitashogakko.onmicrosoft.com（北小学校）
- tokyo-university.onmicrosoft.com（東京大学）
- sakura-highschool.onmicrosoft.com（さくら高等学校）

避けるべき例:
- school123.onmicrosoft.com（学校が特定できない）
- test-school.onmicrosoft.com（テスト用と混同）
```

> ⚠️ **注意**: onmicrosoft.comドメインは後から変更できません。学校名が明確にわかる名称を選択してください。

## 🚀 4. Microsoft 365 Education テナント作成手順

### **4.1 無料ライセンス（Office 365 A1）でのテナント作成**

#### **Step 1: Office 365 Education 公式ページへアクセス**

1. **教育機関向けページにアクセス**
   - URL: https://www.microsoft.com/ja-jp/education/products/office
   - 「**無料で開始**」ボタンをクリック

2. **新しいアカウント作成を選択**
   - 「**新しいアカウントの作成**」を選択
   - 既存アカウントがある場合でも新規テナント用は「新規作成」

#### **Step 2: 教育機関情報の入力**

```markdown
入力項目:
□ 国または地域: 日本
□ 姓名: IT管理者の実名
□ 勤務先メールアドレス: 学校の既存メールアドレス
□ 勤務先電話番号: 学校代表電話番号
□ 学校名: 正式な教育機関名
□ 学校の規模: 適切な規模を選択
```

3. **教育機関認証の確認**
   - Microsoftによる教育機関認証プロセス
   - 数日から1週間程度かかる場合があります

#### **Step 3: 管理者アカウント作成**

```markdown
設定項目:
□ ユーザー名: IT管理者ID（例: admin, it.manager）
□ ドメイン名: [学校名].onmicrosoft.com
□ パスワード: 強力なパスワード（英数字記号組み合わせ）
```

> 💡 **教育機関向けパスワードのベストプラクティス**:
> - 最低12文字以上
> - 大文字・小文字・数字・記号を組み合わせ
> - 学校名や生年月日は避ける
> - 定期的な変更（3-6ヶ月毎）

4. **アカウント情報をメモ**
   ```markdown
   【重要】以下の情報を安全な場所に記録
   - ユーザー名: ________________
   - ドメイン: ________________.onmicrosoft.com
   - パスワード: ________________
   - 作成日: ________________
   ```

#### **Step 4: 本人確認**

1. **電話番号による確認**
   - 学校代表電話番号または管理者携帯番号を入力
   - 「確認コードの送信」をクリック

2. **確認コードの入力**
   - SMSまたは音声通話で受信したコードを入力
   - 「確認」をクリック

#### **Step 5: 教育機関認証待ち**
- 「**準備ができました**」をクリック
- 教育機関認証完了後、Microsoft 365 管理センターにアクセス可能

### **4.2 有償ライセンス購入による本格運用準備**

#### **Step 1: CSP/LSPパートナーの選定**

中・大規模組織向けボリューム ライセンス プログラムは、マイクロソフト認定の Licensing Solution Partner (LSP) よりご購入いただけます

**主要なパートナー種別**
| パートナー種別 | 対象組織 | 特徴 |
|---------------|----------|------|
| **CSP（Cloud Solution Provider）** | 全規模対応 | 月額課金、柔軟な契約 |
| **LSP（Licensing Solution Partner）** | 中・大規模教育機関 | 年契約、ボリューム割引 |

**選定基準**
```markdown
✓ 教育機関向け実績・専門知識
✓ 技術サポート体制
✓ 価格・支払い条件
✓ 導入支援サービス
✓ 継続的な運用サポート
```

#### **Step 2: ライセンスプラン決定**

```markdown
教育機関規模別推奨プラン:

小規模校（～100名）:
- Office 365 A1（無料）+ 必要に応じてA1 for Device

中規模校（100-500名）:
- Microsoft 365 A3（教職員）+ Student Use Benefit
- A1 for Device（共有PC用）

大規模校（500名以上）:
- Microsoft 365 A5（教職員）+ Student Use Benefit
- A3（特定用途）+ A1 for Device（共有PC用）
```

#### **Step 3: 契約・調達手続き**

1. **パートナーからの見積取得**
   - 教職員・学生数の正確な情報提供
   - Student Use Benefit適用条件の確認
   - 導入支援サービスの内容確認

2. **学校内承認プロセス**
   - 予算承認・稟議書作成
   - 契約締結権限者の確認
   - セキュリティポリシーとの整合性確認

3. **契約締結・ライセンス配布**
   - 契約書締結
   - ライセンス情報の受領
   - テナントへのライセンス適用

## 🔧 5. 初期設定と管理センターアクセス

### **5.1 Microsoft 365 Education 管理センターへのアクセス**

#### **教育機関向け管理センターURL**
```markdown
主要アクセスURL:
- Microsoft 365 管理センター: https://admin.microsoft.com
- School Data Sync: https://sds.microsoft.com
- Intune for Education: https://intuneeducation.portal.azure.com
- Office 365 Education ポータル: https://portal.office.com
```

#### **初回アクセス手順**

1. **教育機関認定済みブラウザでアクセス**
   - Microsoft Edge推奨（教育機関向け最適化済み）
   - Chrome、Safari、Firefoxも対応

2. **管理者アカウントでサインイン**
   ```markdown
   ユーザー名: [管理者ID]@[学校名].onmicrosoft.com
   パスワード: 作成時に設定したパスワード
   ```

3. **多要素認証設定（必須）**
   - 初回ログイン後、MFA設定画面が表示
   - Microsoft Authenticatorアプリまたは電話番号で設定
   - 教育機関のセキュリティポリシーに準拠

### **5.2 教育機関向け管理センター画面の確認**

#### **教育機関ダッシュボード概要**
```markdown
教育機関向け管理センターで確認できる項目:
├── 学校情報（テナント名、ドメイン）
├── 教育ライセンス使用状況
├── 教職員・学生数
├── 教育サービス正常性
├── セキュリティ推奨事項（教育機関向け）
├── GIGAスクール対応状況
└── 最近の教育アクティビティ
```

#### **重要な確認ポイント**

1. **学校テナント情報の確認**
   ```markdown
   左上の組織名表示を確認:
   - 正しい学校名が表示されているか
   - onmicrosoft.comドメインが学校名に相応しいか
   - 教育機関として認証されているか
   ```

2. **教育ライセンス状況確認**
   - 「**請求**」>「**製品**」で契約内容確認
   - Student Use Benefit適用状況確認
   - 無料試用版の場合は認証状況確認

3. **教育サービス正常性確認**
   - 「**正常性**」>「**サービス正常性**」で障害情報確認
   - 特に教育サービス（Teams for Education等）の状況確認

## ⚠️ 7. よくある問題とトラブルシューティング

### **7.1 教育機関認証に関する問題**

#### **問題: 教育機関として認証されない**
**症状**: 「教育機関として確認できませんでした」
**原因**: 教育機関データベースに未登録、情報不一致
**解決策**:
1. 正式な教育機関名で再申請
2. 文部科学省認可の証明書類提出
3. Microsoftサポートへ直接問い合わせ
4. CSP/LSPパートナー経由での認証支援

#### **問題: Student Use Benefit が適用されない**
**症状**: 学生向け無償ライセンスが利用できない
**原因**: 一部の教職員のみライセンスをご契約いただいた場合は、学生向け無償ライセンス特典は ご利用いただけません
**解決策**:
1. 全教職員のライセンス契約確認
2. 契約プランの見直し
3. パートナーへの契約条件確認

### **7.2 GIGAスクール対応の問題**

#### **問題: GIGAスクール端末が管理できない**
**症状**: 学校配布端末が Microsoft 365 で管理されない
**原因**: Microsoft 365 GIGA PROMOライセンスの提供終了
**解決策**:
1. Microsoft 365 A1 for DeviceやMicrosoft 365 A1 for Device (Legacy)などの新しいライセンスを購入
2. 既存ライセンスから新ライセンスへの移行
3. デバイス管理ポリシーの再設定

### **7.3 教育データセキュリティの問題**

#### **問題: 学生データの適切な管理ができない**
**症状**: 学生の個人情報が適切に保護されていない
**原因**: 教育機関向けセキュリティ設定の不備
**解決策**:
1. 教育機関向けセキュリティポリシーの設定
2. データ分類・ラベル付けの実施
3. 保護者同意管理の実装
4. 定期的なセキュリティ監査の実施

## 🚨 10. 重要な注意事項

### **10.1 教育機関向けテナント統合制限**

> ⚠️ **重要**: 複数のテナントを統合することはできません。テナント間のライセンスの移動はできません

#### **教育機関でのテナント分離の影響**
```markdown
学校でテナントが分かれた場合の制限:
- 学校間でのデータ共有不可（Teams、SharePoint等）
- ライセンス統合不可
- 統合管理不可
- 学校統廃合時のドメイン移行ダウンタイム発生
```

#### **教育機関での事前計画の重要性**
- 教育委員会全体での統一テナント検討
- 学校統廃合予定の事前把握
- 将来的な学校間連携も考慮した設計

### **10.2 GIGAスクール構想への対応**

第2期GIGAスクール構想におけるICT環境のさらなる充実を支援します。教育委員会や学校は、各ライセンスの特性を理解し、自校のニーズに最適なライセンスを選定することが求められます

#### **GIGAスクール対応計画**
- 端末管理ライセンス（A1 for Device）の検討
- ゼロタッチ展開による効率的な端末配布
- 教育コンテンツ管理体制の構築
